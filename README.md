# tredoc

A fine-tuned Python docstring generator. Submit a function body, get a formatted docstring back.

```bash
curl -X POST http://localhost:8080/model/v1/generate \
  -H "Content-Type: application/json" \
  -d '{"code": "def add(a, b):\n    return a + b", "style": "google"}'
```

---

## Architecture

```
┌─────────────┐     POST /model/v1/generate     ┌────────────────┐
│   Client    │ ──────────────────────────────► │  FastAPI (8080) │
└─────────────┘                                  └───────┬────────┘
                                                         │ OpenAI-compat chat
                                                         ▼
                                                  ┌─────────────────┐
                                                  │  vLLM (8000)    │
                                                  │  tredoc model   │
                                                  └─────────────────┘
```

- **FastAPI** handles request validation and routing. The generate endpoint accepts a code string and a style (`google`, `numpy`, `sphinx`, `plain`) and returns the docstring.
- **vLLM** serves the fine-tuned model via an OpenAI-compatible chat completions API. The FastAPI layer calls it with an `AsyncOpenAI` client.
- **PostgreSQL** stores the scraped function/docstring pairs used during training (not required for inference).

---

## Engineering Decisions

### Model: Qwen2.5-Coder-1.5B-Instruct

Several base models were fine-tuned on the same dataset and evaluated head-to-head using GPT-4o as a judge scoring docstring correctness, completeness, and adherence to the requested style. Qwen2.5-Coder-1.5B-Instruct consistently outperformed the other candidates after fine-tuning, making it the clear choice despite being one of the smaller options.

Fine-tuning used LoRA (rank 16) via [Unsloth](https://github.com/unslothai/unsloth) for memory-efficient training on a single GPU. Adapters target all attention and MLP projection layers. The final artifact is tracked in Weights & Biases as `Qwen2.5-Coder-1.5B-Instruct_20260326_233550`.

### Inference: vLLM

vLLM's PagedAttention scheduler and continuous batching give substantially higher throughput than a naive `model.generate()` loop. Since docstring generation is a high-frequency dev-tooling workload, throughput matters more than raw latency for a single request. The OpenAI-compatible API also means the serving contract is stable and well-understood.

### Prompt Format

Training data followed a structured XML-tagged prompt:

```
Create a docstring for python code following the specifications: <style>{style}</style>
<signature>{signature}</signature>
<body>{body}</body>
```

The signature is extracted from the source at inference time using a regex so callers only need to send the full function text. This matches the format the model was trained on exactly.

## Getting Started

**Prerequisites:** Docker, Docker Compose, NVIDIA GPU with the NVIDIA Container Toolkit installed.

```bash
# Download the model artifact (requires W&B login)
cd tredoc_serving
python app/util/download.py

# Start all services
docker compose up --build
```

{
  "code": "<python function source>",
  "style": "google"
}
```

{
  "docstring": "<generated docstring>"
}
```

### 1. Dijkstra's Shortest Path

**Input:**
```python
def dijkstra(
    graph: dict[str, dict[str, float]],
    source: str,
    target: str | None = None,
) -> tuple[dict[str, float], dict[str, str | None]]:
    dist = {node: float("inf") for node in graph}
    dist[source] = 0.0
    prev: dict[str, str | None] = {node: None for node in graph}
    heap = [(0.0, source)]
    visited: set[str] = set()
    while heap:
        d, u = heapq.heappop(heap)
        if u in visited:
            continue
        visited.add(u)
        if u == target:
            break
        for v, w in graph[u].items():
            alt = d + w
            if alt < dist[v]:
                dist[v] = alt
                prev[v] = u
                heapq.heappush(heap, (alt, v))
    return dist, prev
```

**Model output:**
```
Computes shortest paths from a source node using Dijkstra's algorithm.

Args:
    graph: Adjacency mapping where graph[u][v] is the non-negative edge weight
        from u to v.
    source: The starting node identifier.
    target: If provided, the search halts once this node is settled.
        Defaults to None.

Returns:
    A tuple (dist, prev) where dist maps each node to its shortest distance
    from source and prev maps each node to its predecessor on the shortest path.
```

### 2. Async Retry with Exponential Backoff

**Input:**
```python
async def retry_with_backoff(
    func: Callable[..., Awaitable[T]],
    *args: Any,
    max_retries: int = 3,
    base_delay: float = 1.0,
    max_delay: float = 60.0,
    exceptions: tuple[type[Exception], ...] = (Exception,),
    **kwargs: Any,
) -> T:
    last_exc: Exception | None = None
    for attempt in range(max_retries + 1):
        try:
            return await func(*args, **kwargs)
        except exceptions as exc:
            last_exc = exc
            if attempt == max_retries:
                break
            delay = min(base_delay * 2 ** attempt + random.uniform(0, 1), max_delay)
            await asyncio.sleep(delay)
    raise last_exc
```

**Model output:**
```
Calls an async callable and retries on failure with exponential backoff and jitter.

Args:
    func: The async function to invoke.
    *args: Positional arguments forwarded to func.
    max_retries: Maximum number of retry attempts after the initial call.
        Defaults to 3.
    base_delay: Initial wait time in seconds before the first retry.
        Defaults to 1.0.
    max_delay: Upper bound on the computed delay in seconds. Defaults to 60.0.
    exceptions: Exception types that trigger a retry. Defaults to (Exception,).
    **kwargs: Keyword arguments forwarded to func.

Returns:
    The return value of func on the first successful invocation.

Raises:
    Exception: Re-raises the last exception if all retry attempts are exhausted.
```

### 3. Feature Matrix Builder

**Input:**
```python
def build_feature_matrix(
    records: list[dict[str, Any]],
    numeric_cols: list[str],
    categorical_cols: list[str],
    target_col: str,
    *,
    drop_missing: bool = True,
    normalize: bool = True,
) -> tuple[np.ndarray, np.ndarray, dict[str, LabelEncoder]]:
    df = pd.DataFrame(records)
    if drop_missing:
        df = df.dropna(subset=numeric_cols + categorical_cols + [target_col])
    encoders: dict[str, LabelEncoder] = {}
    for col in categorical_cols:
        le = LabelEncoder()
        df[col] = le.fit_transform(df[col].astype(str))
        encoders[col] = le
    X = df[numeric_cols + categorical_cols].to_numpy(dtype=float)
    y = df[target_col].to_numpy()
    if normalize:
        X = StandardScaler().fit_transform(X)
    return X, y, encoders
```

**Model output:**
```
Converts a list of record dicts into a numeric feature matrix and target vector.

Categorical columns are integer-encoded with LabelEncoder. Numeric columns are
optionally standardized with StandardScaler. Rows with missing values in any
relevant column are dropped when drop_missing is True.

Args:
    records: Raw data as a list of flat dictionaries.
    numeric_cols: Column names to treat as numeric features.
    categorical_cols: Column names to label-encode as categorical features.
    target_col: Column name of the prediction target.
    drop_missing: Whether to remove rows containing NaN in feature or target
        columns. Defaults to True.
    normalize: Whether to apply StandardScaler to the feature matrix.
        Defaults to True.

Returns:
    A tuple (X, y, encoders) where X is the (n_samples, n_features) feature
    matrix, y is the (n_samples,) target array, and encoders maps each
    categorical column name to its fitted LabelEncoder.
```
